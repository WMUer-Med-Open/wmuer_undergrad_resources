# Maintainer Guide

这份文档给仓库维护者使用，用于保持资料库结构清晰、来源可追溯、版权和隐私边界稳定。

## 审核顺序

处理 Pull Request 时建议按以下顺序看：

1. 文件是否放在正确年级和类别目录。
2. 文件类型是否支持。
3. 文件名是否符合小写英文、数字、连字符规则。
4. Markdown 元信息是否说明标题、年级、类别、作者、授权、更新时间和来源。
5. PDF/Word 是否配套同名 Markdown 说明。
6. 是否存在版权风险、隐私风险或医学建议风险。
7. 内容是否足够清晰，能否被后来者复用。

## 建议处理方式

| 情况 | 处理方式 |
|---|---|
| 小格式问题 | 评论指出并请贡献者修改。 |
| 目录放错 | 建议移动到正确年级或类别。 |
| PDF/Word 无说明 | 要求补同名 Markdown 说明。 |
| 来源或授权不清 | 暂不合并，要求补充来源和授权。 |
| 疑似版权或隐私问题 | 不合并，必要时删除相关文件。 |
| 医学表述过度绝对 | 要求改成学习笔记语气并标注更新时间。 |

## 年级结构

不要轻易改变固定结构：

```text
years/<year>/summaries/
years/<year>/experience-posts/
years/<year>/documents/
```

如果某个年级内容增长很多，优先在该类别目录下增加索引表，而不是立即拆出复杂子目录。医学资料早期应保持入口简单。

## 索引维护

每个类别 README 都有资源索引表。新增资料后，建议同步更新对应表格：

```md
| 资料 | 主题 | 格式 | 更新时间 | 说明 |
|---|---|---|---|---|
| example.md | 生理学 | Markdown | 2026-06-05 | 复习框架 |
```

如果资料只有外部链接，不上传文件本体，也可以在说明中标注“外部资源索引”。

## 自动检查

仓库包含 `scripts/check_repository.py` 和 GitHub Actions。自动检查主要覆盖：

- 必需目录是否存在。
- `years/` 下是否只有支持的文件类型。
- PDF/Word 是否放在 `documents/` 并配套同名 Markdown 说明。
- 文件名是否符合规则。
- Markdown 文件是否能用 UTF-8 读取。
- 二进制文档是否超过大小限制。

自动检查不能替代人工审核版权、隐私和医学准确性。

## 紧急移除

如果发现已经合并的文件存在版权或隐私问题：

1. 先从当前分支移除或替换问题文件。
2. 在 Issue 或 PR 中记录处理原因。
3. 如果问题严重，联系仓库所有者评估是否需要清理 Git 历史。
4. 对贡献指南或模板补充相应提醒。
